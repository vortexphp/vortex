# Internationalization

## Translator

**`Vortex\I18n\Translator`** loads PHP arrays from **`lang/{locale}.php`**. Bootstrap constructs it with:

- **`lang/`** path  
- Default locale from **`app.locale`**  
- Fallback from **`app.fallback_locale`**  
- Supported list from **`app.locales`** (comma-separated env **`APP_LOCALES`** in `config/app.php`)

### Static API

| Method | Purpose |
|--------|---------|
| `Translator::get($key, $replace)` | Translate dot key; missing → fallback locale, then the key |
| `Translator::locale()` | Current locale |
| `Translator::supportedLocales()` | Allowed locale codes |
| `Translator::setLocale($locale)` | Switch locale for this request (if supported) |

PHP helper **`trans($key, $replace = [])`** delegates to **`Translator::get`**. Placeholders in strings use **`:name`** replaced from `$replace`.

> **Example — PHP string with placeholder**

```php
$msg = trans('auth.welcome', ['name' => $user->name]);
// lang file: 'welcome' => 'Welcome, :name!',
```

## Twig

Use **`{{ trans('group.key') }}`** or the **`trans`** function (see [Views](views.md)).

> **Example — Twig**

```twig
<p>{{ trans('blog.read_more') }}</p>
<button type="submit">{{ trans('forms.save') }}</button>
```

## Locale middleware

**`App\Middleware\SetLocale`** (registered in **`app.middleware`**) typically reads user preference or **`Accept-Language`** and calls **`Translator::setLocale`** when the value is in **`supportedLocales()`**.

Mirror every user-facing key in **`lang/en.php`** and **`lang/bg.php`** (or whatever locales you configure).
